3D Graphics acceleration not available with ALL IN WONDER RADEON 8500DV

This information applies to the following:

  • AMD Proprietary Linux Driver
  • ALL IN WONDER RADEON 8500DV

The system starts and 2D applications run fine, but 3D graphics will appear unusually slow on ALL IN WONDER RADEON 8500DV using the AMD Proprietary Linux driver.

Applications that absolutely require hardware acceleration may display an error message or simply crash.

Utilities such as glxinfo will report that Direct Rendering (hardware 3D acceleration) is not available and that the "Mesa GLX Indirect" (Software) renderer will be used instead.

The ALL IN WONDER RADEON 8500DV appears to be detected correctly according to /var/log/XFree86.0.log as follows:

(--) fglrx(0): Chipset: "AMD R200 BB (R8500)" (Chipset = 0x4242)
(--) fglrx(0): (PciSubVendor = 0x1002, PciSubDevice = 0x02aa)
(--) fglrx(0): board vendor info: original AMD grafics adapter
...
(II) fglrx(0): board/chipset is supported by this driver (original AMD board)

And yet the following error message appears further along in /var/log/XFree86.0.log:

(EE) fglrx(0): DRIScreenInit failed!
(WW) fglrx(0): ***********************************************
(WW) fglrx(0): * DRI initialization failed!                  *
(WW) fglrx(0): * (maybe driver kernel module missing or bad) *
(WW) fglrx(0): * 2D acceleraton available (MMIO)             *
(WW) fglrx(0): * no 3D acceleration available                *
(WW) fglrx(0): ********************************************* *

This issue has been resolved in AMD Proprietary Linux Driver version 3.14 and later.
